What does this proof demonstrate?

  • You know a valid commitment (secret + amount + blinding) inside a Merkle tree
  • You know the secret preimage of that commitment
  • You can produce a unique nullifier to prevent double-spending
  • All of this without revealing which commitment is yours

2 Generate Witness ?
Witness Generation
The witness is all the data needed to evaluate the circuit: both private inputs (secret, blinding) and public inputs (root, nullifier hash).

The prover uses this to generate a proof. The verifier never sees the private inputs.

The witness contains all circuit inputs. Private inputs stay secret; public inputs go on-chain.

3 Generate Groth16 Proof ?
Groth16 Proof
A succinct proof (192 bytes) that proves all the statements without revealing private inputs.

Uses elliptic curve pairings on the BN254 curve. Constant size regardless of circuit complexity.

The prover computes three elliptic curve points that encode the entire proof.

Proof will appear here...

4 On-Chain Verification ?
Pairing-Based Verification
The verifier checks a bilinear pairing equation:

e(pi_A, pi_B) = e(alpha, beta) * e(L, gamma) * e(pi_C, delta)

This runs on-chain in ~200k gas. Constant time regardless of circuit size.

The smart contract verifies the proof using only the public inputs. No private data needed.
